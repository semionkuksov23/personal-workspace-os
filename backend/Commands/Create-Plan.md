# Command: Create-Plan

## Purpose
Create a detailed implementation plan before making changes.

## Trigger
- User asks for new workflow, refactor, migration, or structural update.

## Steps
1. Clarify target outcome and constraints.
2. Identify impacted files/folders.
3. Identify unknowns and assumptions.
4. Ask critical clarifying questions first.
5. Produce step-by-step plan with:
   - ordered actions
   - file-level change targets
   - risks and mitigations
   - validation approach
6. Wait for explicit plan confirmation before implementation.

## Output Template
- Objective
- Scope
- Steps
- Risks
- Open questions
- Validation checklist
